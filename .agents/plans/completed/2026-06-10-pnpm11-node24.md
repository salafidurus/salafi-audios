# pnpm 11 + Node 24 Upgrade Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade the monorepo to require Node 24 and pnpm 11, updating all engine constraints, `packageManager` field, and CI workflow node versions.

**Architecture:** All changes are configuration-only — no application code changes. The root `package.json` declares engines and `packageManager`; CI workflows pin `NODE_VERSION`; the lockfile must be regenerated under pnpm 11.

**Tech Stack:** pnpm 11, Node 24, GitHub Actions (`actions/setup-node@v6`, `pnpm/action-setup@v6`)

This project uses nvm. Before starting, switch to Node 24 and install pnpm 11.



## Gotchas

- **pnpm 11 lockfile format**: the new lockfile is not backwards-compatible with pnpm 10. Once committed, all contributors and CI must use pnpm 11.
- **`pnpm/action-setup@v6`** in CI reads `packageManager` from `package.json` automatically — no version input needed in the YAML. After updating `packageManager`, CI will auto-pick pnpm 11.
- **nvm users**: after pulling this branch, contributors must run `nvm install 24 && nvm use 24 && npm install -g pnpm@11` before `pnpm install`. The `.nvmrc` file signals the required version; `nvm use` will apply it automatically if they have a shell hook configured.
- **Node 24 native modules**: if any package uses native addons (`.node` binaries), they may need recompilation. This project has no Dockerfiles and the `onlyBuiltDependencies` list is small — check those packages compile cleanly after `pnpm install`.
- **`engines` strict mode**: pnpm enforces `engines` by default. If a dev runs with Node 22 after this change they'll see a warning (not a hard fail unless `engine-strict=true` is in `.npmrc`). Check `.npmrc` for this setting.
